### Finding F-003: GitHub Actions Security Configuration Gaps

**Finding ID:** F-003
**Title:** GitHub Actions Use Mutable References and Lack Integrity Verification
**Severity:** Medium
**Status:** Open

**Framework Mapping:**
- NIST CSF: PR.DS-06, ID.RA-08
- SOC 2: CC6.1, CC8.1

**Criteria:**
Security best practices require pinning action versions to specific commits and verifying action integrity.

**Condition:**
GitHub Actions workflows use mutable tag references (e.g., @v5, @v4) instead of commit SHAs:
- actions/checkout@v5 (9 instances)
- actions/setup-node@v6 (6 instances)
- Multiple other actions using mutable references

**Evidence:**
- Workflow files analysis (Evidence REF: GHA-002)
- 18 unique GitHub Actions identified using mutable references

**Cause:**
Convenience prioritized over security in action version management.

**Effect:**
Potential for supply chain attacks via compromised GitHub Actions, as tags can be moved to malicious commits.

**Recommendation:**
1. Pin all GitHub Actions to specific commit SHAs
2. Implement action integrity verification
3. Regular review and update of action versions
4. Consider using action signature verification
